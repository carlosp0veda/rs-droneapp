# Data Model: SDLC Agents and Orchestrator

## Entities

### Agent
Represents a specialized AI entity within the SDLC process.

- **Role**: (String) e.g., "Specifier", "Planner", "Coder", "Reviewer".
- **SystemPrompt**: (Text) The instructions defining the agent's behavior.
- **ModelConfig**: (JSON) LLM configuration (model name, temperature, etc.).
- **Tools**: (List[String]) List of registered tools this agent can use.

### Workflow
Defines the structure of the SDLC process.

- **Name**: (String) e.g., "Feature-Development", "Bug-Fix".
- **Structure**: (JSON) The nested arrangement of `SequentialAgent`, `ParallelAgent`, and `LoopAgent` wrappers.
- **Configuration**: (JSON) Global settings for the workflow (timeouts, retries).

### Execution (Step/Trace)
A runtime instance of a workflow for a specific feature.

- **FeatureID**: (String) The feature branch or name.
- **CurrentState**: (Enum) `ENUM(NOT_STARTED, SPECIFYING, PLANNING, IMPLEMENTING, VERIFYING, COMPLETED, FAILED)`.
- **History**: (List[Message]) The full conversation history and tool traces.
- **StartedAt**: (Timestamp)
- **UpdatedAt**: (Timestamp)

### Artifact
A versioned output generated by an agent.

- **Type**: (Enum) `ENUM(SPEC, PLAN, CODE, TEST_RESULT, REPORT)`.
- **Path**: (String) Location in the filesystem.
- **Metadata**: (JSON) Information like source agent, execution timestamp, and status.

## State Transitions (SDLC Workflow)

| From | Action | To |
| ---- | ------ | -- |
| `NOT_STARTED` | User triggers `/speckit.specify` | `SPECIFYING` |
| `SPECIFYING` | Agent generates `spec.md` | `SPECIFYING` (Draft) |
| `SPECIFYING` | User approves `spec.md` | `PLANNING` |
| `PLANNING` | Agent generates `plan.md` | `PLANNING` (Draft) |
| `PLANNING` | User approves `plan.md` | `IMPLEMENTING` |
| `IMPLEMENTING` | Coding Agent writes code | `VERIFYING` |
| `VERIFYING` | Tester Agent runs tests | `IMPLEMENTING` (on Fail) or `COMPLETED` (on Pass) |
